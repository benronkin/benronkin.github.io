Check-in Form with Camera Photo

Google Apps Script

SUMMARY

My client needed his contractors to perform onsite check-ins on a tablet using the device camera. I created a web app that enables users to snap their camera photos, upload them to Google Drive, and register their check-ins in a Google Sheet.

Check-in Web App

A Google Apps Script web app contains a custom form. The form contains a name field and a live stream of the camera.

Users fill out their names and click the Take a Picture button. The app grabs a static frame from the video and displays it on the screen.

Users can retake the photo or check in.

Google Drive Folder for Photos

The Google Apps Script has a Google Drive folder to house the photos.

When a user checks in, the script converts their video frame to a PNG image, and saves the file in the folder.

The script automatically names the new file by concatenating the user name from the form with the date and time of the check-in.

Check-in Log

The Google Apps Script works with a Google Sheet log. When a user checks in and a photo file is stored, the script appends the registration into the log.

Each line in the log contains three columns: Date and time of the check-in, the name of the user, and a URL to the image file in the folder.

Managers can easily access the photos by clicking on the URLs.
