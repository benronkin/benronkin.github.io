Unlimited Document Merge

Google Apps Script

SUMMARY

My client needed to merge multiple Google Sheet rows into templated Google Docs. He had many source data tabs and document templates to choose from. I created a Google Apps Script that automates data merge, and scales as more data tabs and templates are created.

Google Docs templates

Users create Google Doc templates with placeholders. The placeholder names match the column names in their Google spreadsheet.

Templates are stored in a main Google Drive folder. The folder contains sub-folders that group the templates by category (e.g. "Trusts").

Users have complete freedom to create as many sub-folders as they need, and have as many templates in each sub-folders as needed. They can always rename sub-folders and templates.

When users launch the sidebar, my automation scans the main folder, and pulls the names of all sub-folders and all of their respective templates.

Custom Google Sheet Sidebar

Users access their Google Sheet that contains the data to merge. The sheet contains multiple tabs, and users navigate to the right tab, and then launch a sidebar from a custom menu.

Users enter the Google Drive folder that contains their Google Docs templates. The automation remembers this folder for future use.

Templates are stored in sub-folders. Users select the right sub-folder from a dropdown menu. Then they select the right document template located in that sub-folder.

Users enter the Google Drive folder in which they want to store their merged document. The automation remembers this folder for future use.

Merged documents

When the user clicks the "Merge" button in the sidebar, the script inspects the current tab the user is on and pulls all the rows.

It duplicates the Google Doc template that the user selected for each row of data.

It replaces each placeholder in the document with the data from the row based on the column header.

Finally, the script saves the document to the destination Google Drive folder that the user selected. It also creates a PDF file and stores it in the same folder.
