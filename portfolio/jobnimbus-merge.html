Document Merge with JobNimbus

Google Apps Script

SUMMARY

My home renovation client needed to pull home owner data from his JobNimbus CRM, and create and populate new Google Docs with that information. I created a Google Apps Script that uses the JobNimbus API to pull customer data and automate document merge jobs.

Get JobNimbus data

A Google Apps Script contains a function that is triggered every ten minutes. The function contacts the JobNimbus API and pulls all the recent jobs. The script stores the address and contact ID of each job.

For every job, the script then triggers a second call to JobNimbus, passing in the contact id. The script stores the contact's name, email, and phone.

Log job data

Once the JobNimbus data is collected, the screen opens a Google Sheet log. It checks that the JobNimbus data isn't already in the sheet, and then appends it to the sheet.

Automated document creation

Next, the script creates a new Google Drive folder and uses the job address as the name of the folder.

The client maintains a set of Google Doc templates inside a Google Drive folder. The client can add, edit, and remove templates as they see fit.

Once the job folder is created, the script automatically copies every document in the template folder into the new job folder.

Doc merge

The document templates contain placeholders for data insertion. When the script duplicates the templates, it takes the JobNimbus data for every job and contact, and pastes them over the placeholders.
